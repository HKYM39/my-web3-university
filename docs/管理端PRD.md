# **管理端（Creator / Admin）PRD**

版本：v1.0  
适用对象：课程作者、平台管理员、开发团队。

---

# **1. 产品目标（管理端）**

- 让课程作者能创建课程、管理课程内容、跟踪收益

- 让管理员能审核课程、管理平台内容

- 让创作者可以将收益质押 AAVE 获得链上收益

管理端包括 Creator（作者）与 Admin（管理员）两类角色。

---

# **2. 用户范围（Persona）**

| 用户类型  | 描述          | 核心需求              |
| ----- | ----------- | ----------------- |
| 课程作者  | 上传内容，期待收入增长 | 创建课程、跟踪收益、质押 AAVE |
| 平台管理员 | 控制平台质量      | 审核课程、管理内容、处理违规    |

---

# **3. 功能需求（FR）**

---

## **3.1 作者端功能（Creator）**

### **3.1.1 课程管理模块**

| 编号       | 功能     | 描述                        |
| -------- | ------ | ------------------------- |
| C-FR-001 | 创建课程   | 输入课程标题、简介、目录、价格           |
| C-FR-002 | 编辑课程内容 | 可修改非链上字段                  |
| C-FR-003 | 提交审核   | 上架课程需管理员审核                |
| C-FR-004 | 查看课程状态 | 审核中 / 上架中 / 驳回            |
| C-FR-005 | 上传视频   | 上传至第三方 CDN（S3/Cloudflare） |

---

### **3.1.2 收益管理模块**

| 编号       | 功能        | 描述               |
| -------- | --------- | ---------------- |
| C-FR-010 | 查看课程收入    | 基于链上转账记录         |
| C-FR-011 | 收益提现      | 可提取 YIDENG / ETH |
| C-FR-012 | 收益质押 AAVE | 可供用户 supply()    |
| C-FR-013 | 查看质押收益    | 显示 APY、aToken 数量 |

### 流程（Creator）

```
查看收益 → 选择资产 → approve → supply → 质押成功 → 查看收益累积
```

---

### **3.1.3 学习数据查看**

| 编号       | 功能           |
| -------- | ------------ |
| C-FR-020 | 查看课程销售量      |
| C-FR-021 | 查看用户学习进度聚合数据 |
| C-FR-022 | 查看奖励发放情况     |

---

## **3.2 管理员端功能（Admin）**

### **3.2.1 课程审核模块**

| 编号       | 功能     | 描述        |
| -------- | ------ | --------- |
| A-FR-001 | 审核课程   | 审核新课程上线   |
| A-FR-002 | 审核结果反馈 | 通过/驳回并附原因 |

---

### **3.2.2 内容管理模块**

| 编号       | 功能        | 描述          |
| -------- | --------- | ----------- |

| A-FR-011 | 首页配置      | 推荐课程配置      |
| A-FR-012 | 学习挖矿奖励池管理 | 控制奖励发放参数    |

---

### **3.2.3 风控与监控**

| 编号       | 功能                |
| -------- | ----------------- |
| A-FR-020 | 异常行为监控（同地址多次失败购买） |
| A-FR-021 | 防刷挖矿监控            |
| A-FR-022 | 链上合约事件日志查询        |

---

# **4. 操作流程（Flow）**

---

## 作者端创建课程流程

```
填写课程信息 → 提交审核 → 管理员审核通过 → 课程上架 → 用户可购买
```

## 管理员审核流程

```
查看待审核课程 → 审核内容与价格 → 通过/驳回 → 返回结果
```

---

# **5. 非功能性需求（NFR）**

| 类型  | 要求                      |
| --- | ----------------------- |
| 可用性 | 后台可用率 ≥ 99%             |
| 安全  | 操作均记录日志                 |
